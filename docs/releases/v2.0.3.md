# Claw-Empire v2.0.3 Release Notes

- Release date: 2026-03-08
- Scope: safe extraction of PR #54's work-phase visibility idea without adopting the risky admin/local-server/system-console portions.

## Highlights

### 1. Task Progress Now Exposes Work Phase Separately from Status

- Added a new nullable `work_phase` field on tasks instead of overloading `status`.
- This keeps the existing task lifecycle intact while exposing the nature of the current work:
  - `api_work`
  - `component_dev`
  - `ui_work`
  - `documenting`
  - `debugging`
- Result: teams can now track "what kind of work is happening" without breaking existing inbox/planned/in_progress/review/done flows.

### 2. Task Creation Can Infer Work Phase from Title Prefixes

- `POST /api/tasks` now infers `work_phase` from leading title tags such as:
  - `[API]`
  - `[UI]`
  - `[コンポーネント]`
  - `[문서]`
  - `[debug]`
  - `[组件]`
  - `[文档]`
  - `[调试]`
- This was intentionally implemented as opt-in inference on creation, not as a replacement for status routing.

### 3. Task Board Can Filter, Display, and Edit Work Phase

- Added a `Work Phase` filter to the task board filter bar.
- Added work-phase badges to task cards so the current implementation focus is visible at a glance.
- Added inline task-card editing for `work_phase`.
- Result: the board now supports both lifecycle tracking and work-shape visibility.

### 4. PATCH Semantics Were Hardened to Avoid Silent Data Loss

- `PATCH /api/tasks/:id` no longer re-infers and overwrites `work_phase` when only the title changes.
- `work_phase` is updated only when explicitly provided.
- Result: manually assigned work-phase values stay stable during title edits.

### 5. Multilingual Support Covers UI Labels and Title-Prefix Inference

- Added localized `work_phase` labels in English, Korean, Japanese, and Chinese across the task-board UI.
- Added Chinese alias support for server-side title-prefix inference so tags such as `[组件]`, `[文档]`, and `[调试]` classify correctly.
- Result: the new work-phase feature behaves consistently across the current multilingual surface.

## PR Reference

- Inspired by `PR #54`, but intentionally re-implemented as a smaller safe subset.
- Not included from PR #54:
  - admin/local-server routes
  - system console and app-list tooling
  - patch auto-apply / auto-commit pipeline
  - task-status enum expansion
  - kickoff / completion minute auto-generation
